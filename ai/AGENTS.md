# Repo Context Pack

This repo is Damien Murphy's personal Astro blog, based on AstroPaper and branded as "stress-tested". It is a static site focused on DevSecOps, platform engineering, CI/CD, containers, security, automation, local tooling, and field notes.

## Stack

- Astro 6, TypeScript, Tailwind CSS v4, pnpm.
- Content lives in `src/content/posts` and `src/content/pages`.
- Main config files:
  - `astro-paper.config.ts`: user-facing site, theme, feature, social, and content settings.
  - `src/config.ts`: resolved config with defaults.
  - `src/types/config.ts`: config type definitions.
- Build and validation commands:
  - `pnpm run lint`
  - `pnpm run format:check`
  - `pnpm run build`

## Content Model

Blog posts use the Astro content collection in `src/content.config.ts`.

Required post frontmatter:

- `title`
- `description`
- `pubDatetime`
- `tags`

Optional post frontmatter:

- `author`
- `modDatetime`
- `featured`
- `draft`
- `ogImage`
- `canonicalURL`
- `hideEditPost`
- `timezone`

Use `pubDatetime` and `modDatetime` exactly. Do not use `pubDateTime`, `pubDate`, `updateddatetime`, or similar variants.

Draft posts are always excluded. Scheduled posts are excluded in production until `pubDatetime - config.posts.scheduledPostMargin`.

## Routing

- Posts are rendered by `src/pages/posts/[...slug]/index.astro`.
- Post URLs are generated by `src/utils/getPostPaths.ts`.
- Subdirectories under `src/content/posts` become part of the URL.
- Home page shows featured posts first, then recent posts from `getSortedPosts`.
- Sorting uses `modDatetime` when present, otherwise `pubDatetime`.

## Site Identity

- Site title: `stress-tested`.
- Author: Damien Murphy.
- Tone: practical, technical, reflective, field-note style.
- About page themes: engineering, security, automation, developer productivity, curiosity, leverage, continuous improvement.
- Avoid generic upstream AstroPaper wording unless explicitly editing theme docs.

## Styling

- Global styles: `src/styles/global.css`.
- Theme tokens: `src/styles/theme.css`.
- Main layout width uses `.app-layout` and `.max-w-app`.
- Light accent: `#006cac`.
- Dark accent: `#ff6b01`.
- Use existing Astro components and Tailwind utility style before introducing new abstractions.

## Important Gotchas

- The upstream README is not fully project-specific; prefer local config and source files over README claims.
- The post template is `src/content/posts/_template.md` and should stay aligned with `src/content.config.ts`.
- `astro-paper.config.ts` may still contain upstream-ish values that need project-specific correction, including `site.url`, `site.description`, `site.timezone`, the mail social URL, and the edit-post GitHub URL.
